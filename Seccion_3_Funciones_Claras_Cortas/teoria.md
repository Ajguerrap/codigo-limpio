# Sección 3: Funciones Claras y Cortas

## Marco Teórico y Fundamentos

- Principio SRP: Una función, una responsabilidad.
- Funciones pequeñas = código más mantenible.
- Framework FIRST: Fast, Independent, Repeatable, Self-validating, Timely.

### Expansión teórica:
- Las funciones cortas facilitan la depuración y el testing.
- Una función debe hacer solo una cosa y hacerla bien.
- Las funciones claras permiten reutilización y reducen errores.
- El límite de líneas es una guía, pero la claridad es la prioridad.
- Separar lógica de presentación y procesamiento.
